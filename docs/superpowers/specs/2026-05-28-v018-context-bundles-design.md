# v0.18 — Context Bundles + Benchmark

**Date:** 2026-05-28
**Status:** Design approved (rolled forward from v0.16 roadmap), ready for plan
**Slice:** v0.18 follow-up to v0.17 blast-radius pack
**Branch:** `v0.18-context-bundles` off `v0.17-blast-radius`

## 1. Goal — the headline selling point

> drag-lint cuts AI assistant per-task token usage on Delphi codebases by an
> order of magnitude, with zero data leaving the machine.

This version turns that claim into a benchable, reproducible CLI command. One MCP
call returns the minimum slice an AI needs to complete a stated task — replacing
~5-10 file reads with one structured payload.

## 2. The command

```
drag-lint context --task "<verb> <qname>" [--db PATH] [--format md|json|raw]
                  [--caller-context N] [--max-callers M] [--no-docs]
                  [--include-class-surface] [--include-impl]
```

### Task syntax

For v0.18, `--task` accepts a verb + qualified name. The verb shapes which blocks
are bundled:

| Verb | Bundle contents |
|---|---|
| `modify` (default) | impl-slice + class-surface + immediate callers + docs |
| `inspect` | class-surface + docs only (no impl, no callers) |
| `refactor` | impl-slice + class-surface + transitive callers depth=2 + docs |
| `delete` | callers up to depth=2 + impact summary + docs |
| `extend` | class-surface + docs + nearest sibling methods (interface signatures) |

If `--task "TFoo.DoBar"` is given without a verb, default to `modify`.

### Output (text / md)

A single Markdown document with H2 sections per block:

```
# Context bundle: modify Foo.TBar.Baz

> Generated by drag-lint v0.18 at 2026-05-28T10:23:45Z
> Token count (estimated): 847
> Replaces ~5 file reads (~6200 tokens)

## Doc
**Summary:** Computes the baz for the given value.
**Param** `value` — input integer, must be > 0
**Returns:** the baz as a string

## Class surface (TBar)
```pascal
TBar = class
public
  function GetBaz(value: Integer): string;
  function Add(A, B: Integer): Integer;
end;
```

## Impl slice
```pascal
function TBar.GetBaz(value: Integer): string;
begin
  Result := IntToStr(value);
end;
```

## Callers (3)
- Caller1.pas:42 in Caller1.Main
  ```
  41: var X: string;
  42:   X := Foo.TBar.GetBaz(5);
  43: end;
  ```
- ... (truncated to --max-callers)
```

### Output (json / raw)

`json` returns the same content as a structured object:
```json
{
  "task": "modify",
  "qname": "Foo.TBar.Baz",
  "generated_at": "2026-05-28T10:23:45Z",
  "token_estimate": 847,
  "blocks": {
    "doc": {...},
    "class_surface": {...},
    "impl_slice": {...},
    "callers": [...]
  }
}
```

`raw` concatenates the source text only (no markdown wrapping) — useful when
piping into prompts.

## 3. Token estimation

Use a simple character-count heuristic: `chars / 3.7` ≈ tokens (per
gpt-3.5/4 averages on technical text). Print the estimate at the top of every
bundle. Heuristic is intentional — exact tokenization is not the headline
claim; "an order of magnitude" is.

## 4. Benchmark suite

New CLI: `drag-lint bench-context [--corpus PATH] [--n 20]`

Picks N random documented methods from the indexed corpus, builds a context
bundle for each, computes:

- bundle tokens (estimated)
- naive baseline: total chars of the source file containing the symbol +
  source files of immediate callers, divided by 3.7
- reduction ratio

Output:
```
Bench: Micronite ORM3 (44169 syms, 795 files), N=20
  Average bundle tokens:    752
  Average baseline tokens:  6840
  Reduction:                9.1x
```

A `--md` flag writes a `bench/context-2026-05-28.md` report for the README.

## 5. Schema impact

**None.** Context bundles compose existing v0.17 features.

## 6. Storage additions

Add helper:
```pascal
function BuildContextBundle(const AQName: string;
  const AVerb: string;
  ACallerContext, AMaxCallers: Integer;
  AIncludeDocs, AIncludeSurface, AIncludeImpl: Boolean): TContextBundle;
```

Where `TContextBundle` is a record with named blocks:
```pascal
TContextBundle = record
  Task:          string;
  QName:         string;
  GeneratedAt:   TDateTime;
  TokenEstimate: Integer;
  Doc:           TParsedDoc;
  ClassSurface:  TArray<TSurfaceLine>;
  ImplSlice:     TArray<TSliceChunk>;
  Callers:       TArray<TReference>;
  ImpactSummary: TArray<TImpactLevel>;  // only for refactor/delete verbs
end;
```

`BuildContextBundle` orchestrates calls to existing finders:
- `GetSymbolDoc(symId)` if AIncludeDocs
- `GetClassSurface(parentQName)` if AIncludeSurface
- `GetSymbolSlice(qname)` if AIncludeImpl
- `FindCallersByNameWithContext(name, ACallerContext)` truncated to AMaxCallers
- `FindTransitiveCallers(name, 2)` for refactor/delete

## 7. CLI / MCP / LSP surfaces

**CLI:**
- `drag-lint context --task "..." ...`
- `drag-lint bench-context ...`

**MCP:**
- `get_context_bundle` — input `{task, qname, ..., db?}`, returns structured JSON.

**LSP:**
- No new capability in v0.18. v0.20 (LSP completion) will consume bundles
  internally for richer suggestions.

## 8. Stop criteria

1. `drag-lint context --task "modify Docs.TDocDemo.GetBaz" --db <t8>` returns
   a Markdown doc with summary + class surface + impl slice (callers may be
   empty on small fixtures — acceptable).
2. `--task "refactor ..."` includes a transitive caller summary.
3. `drag-lint bench-context --corpus C:\Projects\DB\ORM3\COMMON --n 5` prints
   a reduction ratio ≥ 5x on real Micronite code.
4. MCP `get_context_bundle` returns valid JSON.
5. v0.16-v0.17 tests still pass.
6. README has a "Token reduction (v0.18)" section quoting the benchmark.

## 9. Out of scope for v0.18 (carried)

- Multi-symbol bundles (e.g. "modify these three methods together"). v0.19+.
- Smart truncation when the bundle exceeds a token budget. v0.19+.
- Caching bundles across re-runs of the same task. v0.19+.
- Real BPE tokenization (we use char/3.7 heuristic). v0.19+ if needed.
- Bundle-shaped MCP server endpoint that streams blocks. v0.20+.
