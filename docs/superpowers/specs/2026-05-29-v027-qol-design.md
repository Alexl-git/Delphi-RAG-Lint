### 2.2 Test stub generator

`drag-lint generate-test --qname Foo.TBar.Baz [--framework dunitx|dunit] [--format compact|expanded]`

For the given symbol:
1. Look up its signature.
2. Emit a DUnitX test scaffold:

```pascal
[TestFixture]
TFooTBarBazTests = class
public
  [Test]
  procedure Test_Baz_HappyPath;
  [Test]
  procedure Test_Baz_EdgeCases;
end;

implementation

procedure TFooTBarBazTests.Test_Baz_HappyPath;
var
  Subject: TBar;
begin
  Subject := TBar.Create;
  try
    Assert.AreEqual('expected', Subject.Baz(42), 'Baz happy path');
  finally
    Subject.Free;
  end;
end;

procedure TFooTBarBazTests.Test_Baz_EdgeCases;
begin
  // TODO: edge cases
end;
```

MCP tool: `generate_test_stub`.

### 2.3 YADF format integration

`drag-lint format <file> [--yadf-path PATH]`

Shells out to YADF.exe (the user's own Delphi formatter) and writes the formatted output back.

1. Default YADF path: lookup in registry under `HKCU\Software\YADF\ExePath`, else look at `C:\Projects\YADF\Win32\Release\EXE\YADF.exe`, else fail with a usage hint.
2. Spawn synchronously: `YADF.exe "<file>"` (YADF rewrites in place per its convention).
3. Exit code reports success/failure.

Plugin menu entry: `Tools → drag-lint → Format with YADF` — runs YADF against the active editor file.

MCP tool: `format_file`.

## 6. Stop criteria

### Auto-verifiable

1. `drag-lint generate-test --qname Calls.TWidget.Compute` prints a DUnitX scaffold.
2. `drag-lint format <copy of Calls.pas>` runs YADF; original is reformatted; .bak written.
3. T35-T40 still pass.
4. BPL builds.

### Manual

5. In RAD Studio, `Tools → drag-lint → Rename Symbol...` opens the new VCL form (not InputBox).
6. Preview button populates the memo with the dry-run output.
7. Apply button writes the changes.
8. `Tools → drag-lint → Format with YADF` reformats the active editor file.
