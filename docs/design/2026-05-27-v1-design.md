# Delphi-RAG-Lint v1 Design

Status: Draft, awaiting user re-review on return
Author: Alexander Liberov (with Claude/Opus)
Date: 2026-05-27
Workstream: delphi-rag-lint

## Mission

A pure-Delphi, MIT-licensed tool that gives any Delphi project a symbol-aware
index, fuzzy-tolerant lookup, AST-aware retrieval, and structural lint. No
in-process AI; no Python/Node/Rust at runtime. Complementary to commercial
Delphi-native text-RAG tools (which answer "what does this library do" by
LLM synthesis); Delphi-RAG-Lint owns the "where exactly in our code" queries
where determinism and zero hallucination matter.

## Differentiation

| Tool | Source | RAG type | Lint | License | In-Delphi |
|---|---|---|---|---|---|
| **Delphi-RAG-Lint** | this repo | Symbol-exact + fuzzy + BM25 (AST-aware) | Yes | MIT | Yes |
| Commercial Delphi RAG tools | various | Text RAG + LLM | No | Closed | Yes (binary) |
| 3rd-party Delphi lint tools | commercial | — | Yes | Closed | Yes |
| 3rd-party Pascal AST library | FOSS | Parser only | No (lib) | MIT | Yes |

To date there is no FOSS Delphi-native tool that combines symbol-aware RAG with
structural lint. This fills that gap.

## v1 Scope (thin slice of three layers, one shared foundation)

Per brainstorm 2026-05-27: v1 ships a single indexer that all three layers sit
on, plus one demo per layer.

**Shared foundation:**
- Tree-sitter-driven parser for `.pas`, `.dpr`, `.dpk`, `.dfm`
- Symbol model (definitions + references) persisted to a swappable
  `ISymbolStore` — SQLite (via FireDAC) is the v1 default
- AST-aware text chunks indexed by SQLite FTS5 (BM25)
- Fuzzy symbol matcher (Levenshtein + trigram) over the symbol name index

**Three thin-slice demos:**
1. **CLI command** — `drag-lint index <path>` and `drag-lint query` work
2. **RAG / Opus query** — `drag-lint query find-callers --symbol=<X> --json`
   returns a deterministic list with file:line ranges that Opus consumes
3. **Lint rule** — `drag-lint lint --rule=field-by-name-in-loop` flags every
   `FieldByName(...)` call inside a `while|for|repeat` body (the canonical
   anti-pattern already documented in user memory)

**Out of scope for v1, reserved for follow-ups:**
- MCP server wrapper (v1.5)
- Reusable BPL packaging (v2)
- Firebird Embedded `ISymbolStore` implementation (v2; interface ready in v1)
- SQL file indexing (tree-sitter-sql exists but is generic ANSI; Firebird
  specifics deferred)
- Embeddings / semantic vector search (deferred; cheapest later add is to call
  an existing local embedding endpoint, no new RAM)
- Refactor operations (rename, jump-to-def) — v2+

## Data Flow

### Index path
```
Indexer.IndexFolder(path)
  ├── enumerate files matching {*.pas, *.dpr, *.dpk, *.dfm}
  ├── for each file:
  │     ├── hash + mtime check — skip if Store says unchanged
  │     ├── pick parser by extension
  │     ├── parser.Parse(text) → TTSTree
  │     ├── walk AST → emit TSymbol / TReference / TChunk records
  │     └── Store.BeginFileTx(); UpsertFile; UpsertSymbol*; UpsertReference*;
  │         UpsertChunk*; CommitFileTx
  └── report counts (files, symbols, references)
```

### Query path
```
CLI.Dispatch("query find-callers --symbol=X [--json]")
  └── QueryEngine.FindCallers(X)
        ├── try exact symbol match on (qualified_name = X)
        ├── if none — fuzzy match (Levenshtein top-K, trigram top-K) and
        │   return a "did you mean" with candidates ranked by score
        └── else — Store.SelectReferences(symbol_id=found)
        → return TArray<TReference> → format as JSON (machine) or table (human)
```

### Lint path
```
CLI.Dispatch("lint --rule=field-by-name-in-loop")
  └── LintEngine.Run(ruleId)
        ├── load symbols/refs from Store
        ├── for each rule in registry matching ruleId:
        │     └── rule.Check(astQuery, store) → emit TLintFinding records
        └── format findings as text/JSON
```

Tree-sitter queries (S-expression patterns) drive the lint rule matchers
where possible — e.g. `field-by-name-in-loop` is expressible as a query that
finds `(while_statement (..(method_call name: (identifier) @m (#eq? @m
"FieldByName"))..))`. Falls back to programmatic AST walk for rules the query
language can't express.

## Storage Schema (SQLite, draft)

```sql
CREATE TABLE files (
  id          INTEGER PRIMARY KEY,
  path        TEXT NOT NULL UNIQUE,
  mtime_unix  INTEGER NOT NULL,
  sha256      TEXT NOT NULL,
  parsed_at   INTEGER NOT NULL,
  language    TEXT NOT NULL  -- 'delphi13' | 'dfm'
);

CREATE TABLE symbols (
  id              INTEGER PRIMARY KEY,
  file_id         INTEGER NOT NULL REFERENCES files(id) ON DELETE CASCADE,
  parent_id       INTEGER REFERENCES symbols(id) ON DELETE CASCADE,
  kind            TEXT NOT NULL,  -- unit|class|interface|method|proc|func|prop|field|const|var|type|enum_member
  name            TEXT NOT NULL,  -- 'FormShow'
  qualified_name  TEXT NOT NULL,  -- 'MyUnit.TMyForm.FormShow'
  signature       TEXT,           -- 'procedure FormShow(Sender: TObject);'
  modifiers       TEXT,           -- comma-separated: 'override,virtual,public'
  start_line      INTEGER NOT NULL,
  start_col       INTEGER NOT NULL,
  end_line        INTEGER NOT NULL,
  end_col         INTEGER NOT NULL
);
CREATE INDEX idx_symbols_name ON symbols(name);
CREATE INDEX idx_symbols_qname ON symbols(qualified_name);
CREATE INDEX idx_symbols_file ON symbols(file_id);
CREATE INDEX idx_symbols_parent ON symbols(parent_id);

CREATE TABLE refs (
  id          INTEGER PRIMARY KEY,
  symbol_id   INTEGER REFERENCES symbols(id) ON DELETE SET NULL,  -- resolved target
  file_id     INTEGER NOT NULL REFERENCES files(id) ON DELETE CASCADE,
  kind        TEXT NOT NULL,  -- call|type_use|override|inherits|implements|read|write
  name_text   TEXT NOT NULL,  -- the text as written
  start_line  INTEGER NOT NULL,
  start_col   INTEGER NOT NULL,
  end_line    INTEGER NOT NULL,
  end_col     INTEGER NOT NULL
);
CREATE INDEX idx_refs_symbol ON refs(symbol_id);
CREATE INDEX idx_refs_name ON refs(name_text);
CREATE INDEX idx_refs_file ON refs(file_id);

CREATE TABLE chunks (
  id          INTEGER PRIMARY KEY,
  file_id     INTEGER NOT NULL REFERENCES files(id) ON DELETE CASCADE,
  symbol_id   INTEGER REFERENCES symbols(id) ON DELETE SET NULL,
  kind        TEXT NOT NULL,   -- function|class|unit_header|interface_section|implementation_section
  start_line  INTEGER NOT NULL,
  end_line    INTEGER NOT NULL,
  text        TEXT NOT NULL
);

CREATE VIRTUAL TABLE chunks_fts USING fts5(
  text,
  content='chunks', content_rowid='id',
  tokenize='porter unicode61'
);
-- triggers to keep chunks_fts in sync with chunks (insert/delete/update)

CREATE TABLE lint_findings (
  id          INTEGER PRIMARY KEY,
  rule_id     TEXT NOT NULL,
  file_id     INTEGER NOT NULL REFERENCES files(id) ON DELETE CASCADE,
  start_line  INTEGER NOT NULL,
  start_col   INTEGER NOT NULL,
  end_line    INTEGER NOT NULL,
  end_col     INTEGER NOT NULL,
  severity    TEXT NOT NULL,   -- info|warning|error
  message     TEXT NOT NULL,
  run_at      INTEGER NOT NULL
);
CREATE INDEX idx_lint_run ON lint_findings(run_at);

CREATE TABLE schema_meta (
  key   TEXT PRIMARY KEY,
  value TEXT NOT NULL
);
-- ('schema_version', '1')
```

`ISymbolStore` exposes:
- `Migrate()` — applies schema versions
- `BeginFileTx(path) : TFileTxToken`
- `UpsertFile(token, mtime, sha)`
- `UpsertSymbol(token, …)`
- `UpsertReference(token, …)`
- `UpsertChunk(token, …)`
- `CommitFileTx(token)` / `RollbackFileTx(token)`
- Query methods: `FindByExactName`, `FindFuzzy`, `FindCallers(symbolId)`,
  `FindOverrides(symbolId)`, `SearchText(query)`, etc.

## Phasing

### Phase 0 — Toolchain validation (smoke test)
**Goal:** prove the upstream-bindings/tree-sitter-delphi13 pair compiles + loads + parses one `.pas`.

Steps:
1. Build `tree-sitter-delphi13/src/parser.c` into a Win64 DLL via MSVC
   (`vcvars64.bat` from VS2022 BuildTools at
   `C:\Program Files (x86)\Microsoft Visual Studio\2022\BuildTools\`). The
   `tree-sitter` CLI's `build` subcommand can do this, or we invoke `cl.exe`
   directly. Output to `third_party/dll/tree-sitter-delphi13.dll`.
2. Same for `tree-sitter-dfm`.
3. Clone `upstream-bindings/delphi-tree-sitter` into `third_party/delphi-tree-sitter/`
   (vendored, not git submodule — simpler for a first cut; can convert
   later).
4. Write a minimal `phase0_smoke.dpr` that:
   - Loads the DLL via `TTSLanguage.LoadFromFile`
   - Creates `TTSParser`, sets language
   - Parses the file `tests/fixtures/Smoke.pas` (a 5-line unit)
   - Prints the root node's S-expression
5. Build with msbuild + Delphi 13 toolchain. Run. Assert root S-expression
   matches `(source_file (unit_declaration …))` or whatever the grammar names
   it.

**Pass criteria:** smoke binary prints expected root node, exits 0.
**Fail modes:**
- ABI mismatch → bump upstream-bindings's libtree-sitter or rebuild from source
- DLL load failure → check Win64 vs Win32, check DLL not on PATH issue
- Grammar name mismatch in `tree_sitter_delphi13()` exported symbol → adjust
  the grammar's `Cargo.toml` / `tree-sitter.json` or the binding code

### Phase 1 — Foundation
**Goal:** indexer + SQLite store + first `.pas` indexed end-to-end.

Steps:
1. Create `Delphi-RAG-Lint.dproj` for `drag-lint.exe` (console, Win64).
2. Implement `DRagLint.Core.Interfaces.pas` and `DRagLint.Core.Model.pas`.
3. Implement `DRagLint.Storage.SQLite.pas` (FireDAC SQLite TFDConnection,
   migrations, transaction helpers, all CRUD methods).
4. Implement `DRagLint.Parser.TreeSitter.pas` (thin wrapper).
5. Implement `DRagLint.Parser.Delphi13.pas` — walk AST, emit minimal symbol
   set: `unit`, `class`, `method` (procedure/function), `field`, `property`.
   Skip references for now to keep Phase 1 small.
6. Implement `DRagLint.Core.Indexer.pas` — folder walker, dispatch, batched
   writes.
7. CLI subcommand `drag-lint index <path> --db <path.sqlite>`.
8. Smoke fixture: `tests/fixtures/SimpleUnit.pas` with one unit, one class,
   two methods, one property. After indexing, `sqlite3 fixture.sqlite "SELECT
   kind, qualified_name FROM symbols"` returns the expected 5 rows.

**Pass criteria:** end-to-end run on the fixture produces exactly the
expected symbol set. Committed as Phase-1 milestone.

### Phase 2a — RAG demo
1. Implement `DRagLint.Query.Engine.pas`:
   - `FindByExactName(name)` — `WHERE name = ?` and `WHERE qualified_name = ?`
   - `FindCallers(symbolName)` — joins refs to symbols
2. Implement `DRagLint.Query.Fuzzy.pas` — Levenshtein and trigram, applied
   when exact match returns nothing.
3. Implement reference extraction in `Parser.Delphi13.pas` (method calls,
   type usages — limited scope: just enough for the demo).
4. CLI subcommand `drag-lint query find-callers --symbol=X [--json]`.
5. Demo run: index `C:\Projects\DB\ORM3\COMMON`, query
   `--symbol=TBaseForm.AfterShow`, prove the JSON output matches what an
   exhaustive Grep would give (zero false positives, no false negatives).

### Phase 2b — Lint demo
1. Implement `DRagLint.Lint.Engine.pas` with a rule registry.
2. Implement `DRagLint.Lint.Rules.FieldByNameInLoop.pas` — tree-sitter query:
   ```
   (while_statement body: (_ (call_expression
     function: (identifier) @fn (#eq? @fn "FieldByName")))) @hit
   ```
   (and equivalents for `for_statement`, `repeat_statement`)
3. CLI subcommand `drag-lint lint [--rule=ID] [--json]`.
4. Demo run on Micronite — count findings, confirm they match a manual grep
   minus false positives in strings/comments.

### Phase 2c — CLI polish
1. `drag-lint --help`, `drag-lint --version`
2. Stable exit codes (0 = clean, 1 = findings, 2 = error)
3. `--json` output across all commands

### Phase 3 — Public release prep
1. README.md with quickstart and architecture diagram
2. CONTRIBUTING.md
3. Sample `tests/fixtures/` with shipping rule examples
4. CI: GitHub Actions matrix (Win64 only for v1)
5. **User-gated:** create GitHub repo, push.

### Phase 1.5 / 2 — Deferred but designed-for
- MCP server wrapper exposing the same query/lint operations
- Firebird Embedded `ISymbolStore`
- BPL package for IDE/library integration
- `.dfm`-aware queries ("which forms bind to dataset X")
- Additional lint rules (the rest of the user-memory list)

## Risk Register

| ID | Risk | Likelihood | Mitigation |
|---|---|---|---|
| R1 | ABI mismatch between upstream-bindings bindings and our parser.c | Med | Phase 0 detects immediately; rebuild parser.c against upstream-bindings's expected libtree-sitter, or rebuild upstream-bindings against ours. Both are MIT, both are ours. |
| R2 | Compiled DLL not loadable (Win32 vs Win64 confusion) | Low | Build target explicit, verify with `dumpbin /headers`. |
| R3 | tree-sitter-delphi13 grammar gaps cause indexer to miss real symbols | Med | Phase 1 fixture-driven; surface percentage covered; defer edge cases. The "extensions %" accuracy work continues in the tree-sitter-delphi13 workstream and feeds back here. |
| R4 | FireDAC SQLite limitations on FTS5 / virtual tables | Low | FireDAC supports FTS5 via the SQLite driver; verify in Phase 1. |
| R5 | SQLite write throughput too slow for large projects (Micronite has thousands of files) | Low | Per-file transactions, prepared statements, journal_mode=WAL. Benchmark in Phase 1; not blocking. |
| R6 | False positives in lint due to AST query precision | Low | Lint rules return AST ranges; rules are tree-sitter queries so they're inherently AST-aware, not text-based. |
| R7 | Project becomes hard to onboard for other Delphi devs | Med | README + a single `build.bat` that compiles parser.c → DLL and then the .dproj. Keep deps minimal. |
| R8 | Scope creep (embeddings, MCP, BPL all sneak in) | High | Hard rule: v1 is the thin slice. Each follow-up gets its own brainstorm. |

## Open Items

- **GitHub publishing:** account/org choice deferred to user. Local-only until then.
- **C compiler choice:** MSVC vs Embarcadero's bcc64. v1 plan uses MSVC (universally available). Add a `build.bat` switch for bcc64 later if a Delphi-only contributor lacks MSVC.
- **upstream-bindings vendoring strategy:** simple copy in v1; switch to git submodule when v1 is public.
- **Test framework:** DUnitX vs hand-rolled console runner. DUnitX is the de facto Delphi standard but adds a dep. Decision: hand-rolled in Phase 1 (~50 lines), switch to DUnitX only if tests outgrow it.
- **Per-project config:** future `.drag-lint.json` for rule enabling/disabling per project. Out of v1.
